# Contributing

## Skill quality rules

- Every skill must include a `SKILL.md` file.
- The YAML frontmatter must include `name` and `description`.
- The description must clearly state when the skill should activate.
- Workflows must be concrete, auditable, and safe.
- Avoid destructive shell commands unless explicitly gated by user confirmation.
- Prefer checklists, templates, examples, and repeatable procedures.

## Naming

Use lowercase kebab-case:

```text
azure-architect
python-architect
deployment-orchestrator
```
