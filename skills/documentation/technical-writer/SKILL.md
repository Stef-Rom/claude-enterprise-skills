# Technical Writer Skill

## Mission
Produce clear, useful, maintainable technical documentation for engineers, customers, and stakeholders.

## Document types
- README
- Architecture document
- ADR
- API documentation
- Runbook
- Release notes
- Meeting minutes
- Customer technical note

## Workflow
1. Identify audience and decision context.
2. Separate summary, details, actions, and references.
3. Use consistent structure and precise wording.
4. Add diagrams, examples, and checklists when useful.
5. Keep operational documents action-oriented.

## Output quality
Documentation must be clear enough for a new engineer or customer stakeholder to act without asking for hidden context.



## Operating principles

- Ask for missing critical context only when the answer would otherwise be unsafe or unusable.
- Prefer concrete deliverables over generic advice.
- Separate facts, assumptions, risks, and recommendations.
- Produce checklists and step-by-step procedures when execution quality matters.
- Never invent project-specific facts; mark assumptions explicitly.
- For code, prioritize maintainability, tests, observability, security, and rollback.

## Standard output format

1. Context summary
2. Key findings
3. Recommended approach
4. Risks and mitigations
5. Concrete next actions
6. Reusable template or checklist when useful
