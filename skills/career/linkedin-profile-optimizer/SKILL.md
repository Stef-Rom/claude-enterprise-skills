# LinkedIn Profile Optimizer Skill

## Mission
Transform a LinkedIn profile into a recruiter-friendly, keyword-rich, credible professional brand aligned with the target role, CV, and job market.

## Best suited for
- Senior Full Stack Engineer profiles
- Python / React / Azure engineers
- Azure DevOps and deployment automation profiles
- SaaS architecture and multi-tenant platform profiles
- Customer Solution Architect and product expert profiles
- AI-assisted development positioning
- French and English LinkedIn profiles

## Core objectives
1. Make the profile searchable by recruiters using role-specific keywords.
2. Clarify positioning in the first 5 seconds: title, value, stack, business impact.
3. Align LinkedIn with the CV without copying it word-for-word.
4. Convert responsibilities into proof-backed achievements.
5. Improve credibility through Featured items, projects, certifications, recommendations, and activity.

## Required inputs
When available, use:
- Current LinkedIn profile text or screenshots.
- Current CV.
- Target role or job offer.
- Target geography and language.
- Preferred tone: executive, technical, approachable, consultant, or startup.
- Key technologies and achievements.

Do not invent employers, certifications, metrics, clients, or degrees. If information is missing, mark it as an assumption or propose placeholder wording.

## Workflow

### 1. Profile audit
Evaluate:
- Headline clarity
- About section strength
- Experience structure
- Keyword coverage
- Technical credibility
- Business impact
- Recruiter search optimization
- Consistency with CV
- Visual and Featured section opportunities

### 2. Positioning
Define:
- Primary target title
- Secondary target titles
- Value proposition
- Differentiators
- Technical stack
- Business domains
- Customer-facing strengths

### 3. Keyword strategy
Extract and prioritize keywords by category:
- Role keywords: Software Engineer, Full Stack Developer, Solution Architect, DevOps Engineer, Cloud Engineer.
- Cloud keywords: Azure, Azure DevOps, App Service, Cosmos DB, Key Vault, Entra ID, Application Insights.
- Backend keywords: Python, FastAPI, Flask, APIs, MongoDB, SQL, asynchronous processing.
- Frontend keywords: React, TypeScript, Vite, UI architecture, performance.
- DevOps keywords: CI/CD, QA automation, rollback, release management, deployment automation.
- AI keywords: AI-assisted development, prompt engineering, Claude, ChatGPT, Copilot, Cursor, agents.
- Business keywords: SaaS, customer integration, technical consulting, requirements analysis, product expertise.

### 4. Rewrite sections
Produce optimized versions for:
- Headline
- About
- Experience entries
- Featured section
- Skills list
- Services / Open to work positioning
- Banner text
- Creator mode topics if relevant

### 5. Recruiter conversion review
Review the final profile from:
- HR recruiter perspective
- Engineering manager perspective
- CTO / Head of Engineering perspective
- ATS / LinkedIn search perspective

## Writing rules
- Prefer clear, credible, specific wording over buzzwords.
- Use first person only in the About section when it improves authenticity.
- Use quantified achievements when evidence exists.
- Keep the headline keyword-rich but readable.
- Keep the About section scannable with short paragraphs.
- Avoid exaggerated claims such as guru, ninja, rockstar, visionary, or world-class unless explicitly justified.
- In French profiles, avoid literal translations from English; use natural recruiter vocabulary.
- In English profiles, prefer concise and impact-oriented wording.

## Suggested headline formulas
- Role + Stack + Domain + Impact
- Seniority + Core Expertise + Business Context
- Technical Role + Cloud/AI Differentiator + Customer Impact

Examples:
- Full Stack Developer Python / React | Azure Cloud & DevOps | SaaS Platforms | AI-Assisted Development
- Software Engineer | Python, React, Azure, Cosmos DB | Deployment Automation & Customer Solutions
- Azure / Python / React Engineer | SaaS Architecture | CI/CD, QA Automation & AI-Assisted Delivery

## About section formula
1. One-line positioning.
2. Technical strengths.
3. Business/customer impact.
4. Differentiators.
5. Target roles or collaboration focus.

## Experience formula
For each role:
- Scope: product, platform, clients, team, environment.
- Actions: designed, developed, automated, integrated, optimized, supported.
- Impact: performance, reliability, delivery speed, customer satisfaction, production stability.
- Stack: concise technical environment.

## Quality bar
The final LinkedIn profile must be searchable, credible, aligned with the CV, and strong enough to support applications for senior technical roles.

## Operating principles

- Ask for missing critical context only when the answer would otherwise be unsafe or unusable.
- Prefer concrete deliverables over generic advice.
- Separate facts, assumptions, risks, and recommendations.
- Produce checklists and step-by-step procedures when execution quality matters.
- Never invent project-specific facts; mark assumptions explicitly.
- For code, prioritize maintainability, tests, observability, security, and rollback.

## Standard output format

1. Context summary
2. Key findings
3. Recommended approach
4. Risks and mitigations
5. Concrete next actions
6. Reusable template or checklist when useful
